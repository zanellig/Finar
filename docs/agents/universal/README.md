# Universal Agent Guidance

Agent guidance for language-agnostic defaults.

Use them in this order:

1. `docs/agents/universal/core-engineering-principles.md` for any task.
2. `docs/agents/universal/feature-delivery-playbook.md` for new features, major enhancements, or planned refactors.
3. `docs/agents/universal/bugfix-playbook.md` for defects, regressions, audit findings, and production incidents.

Project-local conventions always win. These docs are the fallback when a project does not already define a stronger pattern.
